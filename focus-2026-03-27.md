## 1. 🚨 CMS Publishing Blocked by Cloudflare 524 Timeout

**Source:** [#doc_party](https://yourcounterpart.slack.com/archives/doc_party) · March 25, 2026

Michelle Sipe is hitting a Cloudflare Error 524 (timeout) when attempting to publish any document where `Draft = Yes`. This is actively blocking two forms from reaching QA, and it's time-sensitive because the new `DocumentationContext` keys can't be used until the CMS is published successfully.

Paula Vazquez outlined the required order of operations for new keys (draft → validate → publish → restart pods → undraft → validate → publish), but the timeout is preventing completion.

**Why it matters:** QA is blocked waiting on these documents. If unresolved, it will delay the release of features dependent on the new DocumentationContext variables.

## 2. 🔒 LiteLLM Python Supply Chain Attack — Action Required

**Source:** [#ai-security](https://yourcounterpart.slack.com/archives/ai-security) · [LiteLLM Supply Chain Attack writeup](https://www.comet.com/site/blog/litellm-supply-chain-attack/)

Chad Nierenhausen flagged a significant Python supply chain attack targeting LiteLLM. Two actions are needed:

1. Audit all codebases (direct and transitive dependencies) to confirm LiteLLM is not in use.
2. Consider rotating any API keys that could have been exfiltrated if any exposure exists.

**Why it matters:** If any Counterpart service or dependency pulls in LiteLLM, sensitive API keys (LLM providers, internal services) may be compromised. This needs a fast audit.
